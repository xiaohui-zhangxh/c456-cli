# c456-sync 对外正文（C456 镜像层）

`c456-sync/` 中的正文会与 C456 `intake` 正文一致，读者包含站外陌生访客。须与 `wiki/` 的「笔记、线索、本地路径」写法区分。

## 1. 标题：只在 Frontmatter

- **禁止**在正文开头使用 `#` 一级标题重复 `c456-title`（或 tool 标题）。C456 页面标题来自元数据，正文再写会 **重复展示**。
- 正文从 **二级标题 `##`** 起，或从导语段落 / 配图起。

## 2. 禁止出现在对外正文中的内容

以下内容视为 **对内/草稿/制作备忘**，不得写入 `c456-sync/` 正文（含上行用的 `--body-file`）：

- 章节名如「配图（对照页首屏）」「以下为截图」等 **工序标签**；可改为直接呈现信息，用 **一句公众导语** 或 **加粗 + 外链 + 图** 完成「图说」。
- **标签式总括标题**：如 `## 一句话结论`、`## TL;DR`、`## 核心观点`（小节里只有一句判词的那种）。**判断句写进第一节正文**（首段或次段），`##` 留给有展开内容的主题（产品名、对比轴、具体问题等）。
- **视口、截图日期、截图命令**（如 `c456 screenshot`）、**素材库 ID 文案**（如「素材 c456:asset/63」）、**仓库相对路径**（如 `raw/tools/...`、`c456-sync/...`、`wiki/...`、`output/...`）。
- 「与本地镜像一致」「参见本仓库信号」等 **仅仓库读者** 能理解的指代。
- 竞品索引里 **仅本地可用的链接**（如 `../signal/...md`）；对外只保留 **https://c456.com/intakes/<id>** 或公开官网 URL。
- **来源块前**不要单独插入整行 `---`（会渲染为横线）；与上一节用空行或自然段落收束即可。

说明：Markdown 图片语法 `![alt](url "title")` 里 **title** 字段若需保留 `c456:asset/<id>` 供平台识别，可以保留；**正文中不要再用自然语言解释 asset ID**。

## 3. 配图：媒体级呈现

推荐形态（示例）：

```markdown
*[Logto](https://logto.io/) — 官网首页。*

![Logto 官网首页](https://c456.com/our-assets/.../....png "c456:asset/63")

*[Clerk](https://clerk.com/) — 官网首页。*

![Clerk 官网首页](https://c456.com/our-assets/.../....png "c456:asset/66")
```

- 每张图前用 **一行斜体或加粗 + 外链** 作公众图说即可。
- **不要**单独开「配图」二级标题专门描述拍摄过程。

## 4. 与 wiki/ 的分工

- **`wiki/`**（含 `wiki/sources/`）可保留 Wikilink、`[[...]]`、本地路径、调研笔记、`来源：C456 收录` 等，便于 Obsidian 内导航。
- **`c456-sync/`** 在发布前应 **自检一遍**：假设读者从未打开本仓库，是否仍自洽、专业、无内部噪音。
